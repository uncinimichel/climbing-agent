📚 multi-pitch.com "Live" — Knowledge Base

The reference brain for the project. Start here, then follow the map below. This folder is documentation, not code — it explains what we're building, why, and how the pieces fit. Keep it truthful: separate the vision (where we're going) from the current state (what actually runs today).

One-line mission: turn a static climbing-guidebook library into a Dynamic, Predictive Trip Decision Engine — one question-and-answer flow that replaces the tedious multi-app logistics of planning a climbing trip.

How to use this folder

  • New here / an AI agent picking up the project? Read in this order:

vision/mission.md → architecture/current-state.md → architecture/overview.md.

  • Building a feature? Find its layer in architecture/phases.md, then check

data/schemas.md and operations/ before writing code.

  • Making a non-obvious call? Log it in roadmap/decisions.md.
  • Keep it honest. If a doc describes something aspirational, label it

(planned). Today only a slice of the vision is live — see current-state.md.
