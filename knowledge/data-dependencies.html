Where the source of truth lives

Your repo holds two parallel data worlds that never touch. Each has its own place to add data, its own weather source, and its own idea of what a "climb" is. This map shows what's hand-authored (the real sources of truth), what's generated, and — at the bottom — exactly where to add a new climb or venue.

Decided — #27: one authored db/corpus.json ↗ is now the single source of truth (curated status:publish + seeded status:draft). The source of truth — db/corpus.json (#27)

Add a climb or crag

  1. Add an areas[] entry (crag with lat/lon, rock, aspect) and/or a routes[] entry to db/corpus.json ↗. Coordinates live here now — no GAZETTEER.
  2. Set status: "publish" for verified data, status: "draft" for unverified. Tags (disciplines, features, hazards) are values from tag-spec.json.
  3. Or bulk-seed from multi-pitch.com: re-run python3 db/tools/build_corpus.py (adds draft rows to review).
→ One file. Loads into Postgres later unchanged. Curated list = filter status=publish.
Trips are derived selections

Plan a trip

  1. Make a trip.json that references area IDs from the corpus — never copies crag facts.
  2. Add only the trip overlay: who's going, from which airport, this-trip priority, the date window.
  3. Compose as many trip lists as you like from the one corpus.
→ Splits crag facts (corpus) from trip choices (overlay) that venues.json used to conflate.
